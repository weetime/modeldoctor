# Unified Run Model Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Collapse three separate persistence paths (`LoadTestRun`, `BenchmarkRun`, ephemeral E2E) into a single `Run` table with `kind` discriminator and `tool / scenario / mode / driverKind` four-tuple, plus a new `Connection` shell table and a `Baseline` shell table — establishing the data foundation for #43–#46, #54, #55, #57, and the rest of Phase F.

**Architecture:** One `Run` Prisma model holds every test execution row, kind-agnostic. A new `RunRepository` owns persistence; a new `RunService` exposes read APIs (`GET /api/runs`, `GET /api/runs/:id`); existing `/api/benchmarks`, `/api/load-test`, `/api/e2e-test` controllers keep their current request shapes but persist through `RunRepository` instead of their own dedicated tables. `Connection` and `Baseline` are added as shells (basic CRUD for the former, FK-only for the latter) so #37 and #43 don't need destructive migrations later.

**Tech Stack:** NestJS 10, Prisma ORM, Zod (via `packages/contracts`), PostgreSQL 16, Vitest 2 (`apps/api/vitest.config.mts` — must stay `.mts`).

**Pre-prod policy (per memory):** No compat shims. `LoadTestRun` and `BenchmarkRun` get dropped outright in the migration. The dev DB for this worktree is wiped — that's expected.

**Branch:** Stay on `feat/regression-suite` (long-lived multi-phase branch per project memory). Do not cut a sub-branch.

**Worktree DB isolation (per memory):** This plan introduces a destructive Prisma migration. Give this worktree its own Postgres database before doing anything. Other worktrees stay on `modeldoctor`; this one moves to `modeldoctor_run_unify`.

**Constraints to respect:**
- `apps/api/tsconfig.json` must not set `incremental: true`.
- Vitest config files in `apps/api/` must stay `.mts`.
- `apps/api` uses vitest@2; `apps/web` uses vitest@1 — do not unify.
- `apps/api/tsconfig.json` `include` must stay narrow (`["src/**/*"]`).
- Conventional-commit prefixes; one logical change per commit; explicit `git add <files>`; commit body ends with `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>`.

- [ ] **Step 2: Add `Run` model**

Append:

```prisma
model Run {
  id           String  @id @default(cuid())
  userId       String? @map("user_id")
  connectionId String? @map("connection_id")

  // Discriminator + four-tuple
  kind       String // 'benchmark' | 'e2e'
  tool       String // 'guidellm' | 'genai-perf' | 'vegeta' | 'e2e' | 'custom'
  scenario   Json   // dataset, concurrency, length distribution, model, etc.
  mode       String // 'fixed' | 'ramp-up' | 'throughput' | 'sla-target' | 'correctness'
  driverKind String  @map("driver_kind") // 'local' | 'k8s'

  // Identification
  name        String?
  description String? @db.Text

  // Lifecycle
  status        String  @default("pending") // 'pending' | 'submitted' | 'running' | 'completed' | 'failed' | 'canceled'
  statusMessage String? @db.Text  @map("status_message")
  progress      Float?

  // Driver bookkeeping (replaces benchmark_runs.job_name)
  driverHandle String? @map("driver_handle") // 'subprocess:1234' or 'namespace/jobName'
  apiKeyCipher String? @map("api_key_cipher") // moved here from benchmark_runs

  // Snapshots
  params          Json   // full create payload (auditing)
  canonicalReport Json?  @map("canonical_report") // adapter-normalized — populated by #53 onward
  rawOutput       Json?  @map("raw_output")
  summaryMetrics  Json?  @map("summary_metrics") // p50/p95/p99/error_rate/ttft for query-time filtering
  serverMetrics   Json?  @map("server_metrics")  // for #60

  // Cross-references (FK targets are nullable until #43–#56 wire them up)
  templateId      String? @map("template_id")
  templateVersion String? @map("template_version")
  parentRunId     String? @map("parent_run_id")
  baselineId      String? @map("baseline_id")

  logs String? @db.Text

  createdAt   DateTime  @default(now())  @map("created_at")
  startedAt   DateTime?                  @map("started_at")
  completedAt DateTime?                  @map("completed_at")

  user       User?       @relation(fields: [userId], references: [id], onDelete: SetNull)
  connection Connection? @relation(fields: [connectionId], references: [id], onDelete: SetNull)
  parent     Run?        @relation("RunParent", fields: [parentRunId], references: [id], onDelete: SetNull)
  children   Run[]       @relation("RunParent")
  baseline   Baseline?   @relation(fields: [baselineId], references: [id], onDelete: SetNull)

  @@index([userId, createdAt])
  @@index([kind, status])
  @@index([connectionId])
  @@index([parentRunId])
  @@index([baselineId])
  @@map("runs")
}
```

describe("RunRepository", () => {
  let repo: RunRepository;

  parentRunId?: string;
  userId?: string;
  search?: string;
  cursor?: string;
  limit?: number;

      data: {
        userId: input.userId ?? null,
        connectionId: input.connectionId ?? null,
        kind: input.kind,
        tool: input.tool,
        scenario: input.scenario,
        mode: input.mode,
        driverKind: input.driverKind,
        params: input.params,
        name: input.name ?? null,
        description: input.description ?? null,
        apiKeyCipher: input.apiKeyCipher ?? null,
        templateId: input.templateId ?? null,
        templateVersion: input.templateVersion ?? null,
        parentRunId: input.parentRunId ?? null,
        baselineId: input.baselineId ?? null,
      },
    });
  }

  findById(id: string): Promise<PrismaRun | null> {
    return this.prisma.run.findUnique({ where: { id } });
  }

  async list(
    input: ListRunsInput
  ): Promise<{ items: PrismaRun[]; nextCursor: string | null }> {
    const limit = Math.min(input.limit ?? 20, 100);
    const where: Prisma.RunWhereInput = {};

    if (input.status) where.status = input.status;
    if (input.connectionId) where.connectionId = input.connectionId;
    if (input.parentRunId) where.parentRunId = input.parentRunId;
    if (input.userId) where.userId = input.userId;
    if (input.search)
      where.OR = [
        { name: { contains: input.search, mode: "insensitive" } },
        { description: { contains: input.search, mode: "insensitive" } },
      ];

    const items = await this.prisma.run.findMany({
      where,
      orderBy: [{ createdAt: "desc" }, { id: "desc" }],
      take: limit + 1,
      ...(input.cursor && { cursor: { id: input.cursor }, skip: 1 }),
    });

    const hasMore = items.length > limit;
    const sliced = hasMore ? items.slice(0, limit) : items;
    return {
      items: sliced,
      nextCursor: hasMore ? sliced[sliced.length - 1].id : null,
    };
  }

  update(id: string, input: UpdateRunInput): Promise<PrismaRun> {
    return this.prisma.run.update({
      where: { id },
      data: input as Prisma.RunUpdateInput,
    });
  }

  delete(id: string): Promise<PrismaRun> {
    return this.prisma.run.delete({ where: { id } });
  }

import type { Run, ListRunsQuery, ListRunsResponse } from "@modeldoctor/contracts";
import type { Run as PrismaRun } from "@prisma/client";

@Injectable()
export class RunService {
  constructor(private readonly repo: RunRepository) {}

  async findById(id: string): Promise<Run | null> {
    const row = await this.repo.findById(id);
    return row ? toContract(row) : null;

  async create(userId: string, input: CreateConnection): Promise<Connection> {
    const row = await this.prisma.connection.create({
      data: {
        userId,
        name: input.name,
        baseUrl: input.baseUrl,
        apiType: input.apiType,
        prometheusUrl: input.prometheusUrl ?? null,
        serverKind: input.serverKind ?? null,
      },
    });
    return toContract(row);
  }

  async list(userId: string): Promise<ListConnectionsResponse> {
    const rows = await this.prisma.connection.findMany({
      where: { userId },
      orderBy: { createdAt: "desc" },
    });
    return { items: rows.map(toContract) };
  }

  async findOwned(userId: string, id: string): Promise<Connection> {
    const row = await this.prisma.connection.findUnique({ where: { id } });
    if (!row) throw new NotFoundException(`Connection ${id} not found`);
    if (row.userId !== userId) throw new ForbiddenException();
    return toContract(row);
  }

  async update(userId: string, id: string, input: UpdateConnection): Promise<Connection> {
    await this.findOwned(userId, id);
    const row = await this.prisma.connection.update({
      where: { id },
      data: {
        ...(input.name !== undefined && { name: input.name }),
        ...(input.baseUrl !== undefined && { baseUrl: input.baseUrl }),
        ...(input.apiType !== undefined && { apiType: input.apiType }),
        ...(input.prometheusUrl !== undefined && { prometheusUrl: input.prometheusUrl }),
        ...(input.serverKind !== undefined && { serverKind: input.serverKind }),
      },
    });
    return toContract(row);
  }

  async delete(userId: string, id: string): Promise<void> {
    await this.findOwned(userId, id);
    await this.prisma.connection.delete({ where: { id } });
  }
}

Pseudocode for the new `run()` method:

```typescript
async run(input: E2ETestRequest, userId?: string): Promise<E2ETestResponse> {
  const created = await this.runs.create({
    userId,
    kind: "e2e",
    tool: "e2e",
    scenario: { probes: input.probes, paths: input.pathOverrides ?? {} },
    mode: "correctness",
    driverKind: "local",
    params: input as unknown as Prisma.InputJsonValue,
  });

  await this.runs.update(created.id, { status: "running", startedAt: new Date() });

  try {
    const results = await this.executeProbes(input);
    const allPassed = results.every((r) => r.passed);
    await this.runs.update(created.id, {
      status: allPassed ? "completed" : "failed",
      completedAt: new Date(),
      rawOutput: { results } as Prisma.InputJsonValue,
      summaryMetrics: {
        total: results.length,
        passed: results.filter((r) => r.passed).length,
        failed: results.filter((r) => !r.passed).length,
      } as Prisma.InputJsonValue,
    });
    return { runId: created.id, results };
  } catch (err) {
    await this.runs.update(created.id, {
      status: "failed",
      statusMessage: err instanceof Error ? err.message : String(err),
      completedAt: new Date(),
    });
    throw err;
  }
}
```
